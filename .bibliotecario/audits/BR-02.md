#### Resumen ejecutivo
`i18n.js` es el motor de internacionalización central del ecosistema CPII. Provee soporte para 4 idiomas (Portugués, Español, Inglés, Francés) mediante un sistema de diccionarios JSON y una lógica de manipulación del DOM basada en atributos `data-i18n`. 
Su versión actual (2.9.0) refleja una evolución constante, con inyecciones de claves específicas para módulos como `webinars.html`, `simulator.html` y `access-form.html`.

#### Contexto de repo
Estado: activo — archivo crítico que actúa como entry-point global.
Dependencias de entrada:
- Sincronización con `localStorage('cpii_lang')` para persistencia de idioma.
- Selector de idioma con ID `lang-selector`.
Dependencias de salida:
- Inyecta contenido en todos los archivos HTML del proyecto (`index.html`, `access-form.html`, etc.).

#### Decisiones clave
- **Persistencia de Estado:** Uso de `localStorage` para que la preferencia de idioma sobreviva a la navegación entre archivos HTML planos.
- **Seguridad (Parche v2.1.0):** Diferenciación entre `innerText` (vía `data-i18n`) y `innerHTML` (vía `data-i18n-html`) para mitigar riesgos de XSS.
- **Flexibilidad de Atributos:** Extensión del motor para manejar `placeholders` y el primer `option` de los selectores mediante `data-i18n-placeholder`.
- **Arquitectura de Diccionario:** Estructura plana por idioma, agrupada lógicamente por prefijos (`nav.*`, `form.*`, `team.*`), facilitando la expansión modular.

#### Arqueología de Marca
- Las traducciones no son literales; adaptan el léxico según el mercado (ej. "Aceder" en PT vs "Acceder" en ES).
- Fuerte enfoque institucional en el Changelog, documentando "hotfixes" y "vibe updates" para webinars y secciones legales.

#### Potenciales conflictos y refactors sugeridos (para Agentes de dirección)
- **Tamaño del Archivo:** Con ~1700 líneas y creciendo, el diccionario único escalará mal. Convendría evaluar la carga dinámica de fragmentos JSON por módulo (ej. `webinars.pt.json`) para reducir la carga inicial en la landing.
- **Duplicidad de Claves:** Se detectan claves duplicadas o prefijos muy similares (`team.*` vs `team.hero.*`) que podrían simplificarse.
- **Lógica de Tracking (Ref):** Como sugirió Cursor en BR-01, la lógica de captura de `ref` que actualmente vive dispersa en los HTML podría centralizarse aquí como un módulo utilitario o un `tracking.js` independiente.

#### Impacto en chasis
`i18n.js` constituye el **cerebro de contenido** del chasis. Define no solo qué se dice, sino cómo persiste la experiencia del usuario a través del cambio de idioma. Es el puente técnico que permite que un sitio estático de múltiples páginas (MPA) se comporte con la coherencia de idioma de una SPA.

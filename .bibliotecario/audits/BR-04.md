#### Resumen ejecutivo
`access-form.html` es el componente de conversión crítica del ecosistema. Implementa un formulario de registro multicapa con validación en tiempo real, asignación automática de gestores según el país y persistencia en Firestore. Además, dispara notificaciones push a Telegram mediante una Cloud Function para alertar al equipo comercial.

#### Contexto de repo
Estado: unificado — ahora utiliza el chasis compartido (`design-system.css`, `tracking.js`).
Dependencias de entrada:
- `i18n.js` (textos dinámicos).
- `utils/tracking.js` (captura de referente).
- `js/design-system.js` (config Tailwind).
- `css/access-form.css` (estilos específicos).
Dependencias de salida:
- Firebase Firestore (colección `leads`).
- Google Sheets Webhook (backup).
- Telegram Cloud Function (alertas móviles).

#### Decisiones clave
- **Asignación de Gestores (BR-04 [SEC-07]):** Uso de un mapa estático (`CLUB_MANAGER_MAP`) para derivar la responsabilidad del lead según el país de residencia.
- **Modo Stealth (Embajadores):** Lógica que inyecta opciones "fantasma" en el selector de embajadores si el `ref` de la URL es válido pero no está en el HTML plano, bloqueando el campo para evitar cambios accidentales.
- **Cumplimiento RGPD:** Inclusión de consentimiento explícito en la metadata del lead antes de la persistencia.

#### Arqueología de Marca
- Evolución táctica: La versión 1.4.1 incluye correcciones de usabilidad móvil (padding dinámico y jerarquía tipográfica) derivadas de diagnósticos previos.

#### Potenciales conflictos y refactors sugeridos (para Agentes de dirección)
- **Hardcoded Config:** Las credenciales de Firebase están expuestas en el frontend. Aunque es común en SPAs/MPAs estáticas, se recomienda endurecer las reglas de seguridad de Firestore (`firestore.rules`) para que solo se permitan `create` y no `read/list`.
- **Lógica de Validación:** Se podría derivar a un script externo (`js/form-logic.js`) para reducir el tamaño del HTML y facilitar el testing unitario de la validación.

#### Aprendizajes extraídos
- [ESTRUCTURAL] La combinación de Firestore para persistencia y Telegram para notificaciones push crea un CRM ligero y altamente reactivo para proyectos de captación de leads sin servidor backend complejo.
